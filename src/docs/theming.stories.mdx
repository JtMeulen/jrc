# Theming

Out of the box JRC has a default color schema to which all components and styling classes adhere. These components and classes are setup with theming in mind.
JRC offers a range of public CSS variables, which can be overwritten at the root of your project in order to create your own custom theme.

## Creating your theme

Introduce a css file at the root of your project in which you will set new colors for the available public CSS classes. This does not necessarily need to be done on the root, you could even add themes per feature or per component.

```css
body {
  --jrc-color-primary: hotpink;
  --jrc-color-secondary: limegreen;
  /* etc */
}
```

## Available theming variables

Below you the all available public CSS variables with their default values. Simply copy this file and change it's values to set your own theme.

```css
body {
  --jrc-color-primary: #29339b;
  --jrc-color-primary-hover: #3440c2;
  --jrc-color-primary-active: #22297a;
  --jrc-color-secondary: #74a4bc;
  --jrc-color-secondary-hover: #87bfdb;
  --jrc-color-secondary-active: #6790a5;
  --jrc-color-tertiary: #b6d6cc;
  --jrc-color-tertiary-hover: #cdefe5;
  --jrc-color-tertiary-active: #9ebab1;
  --jrc-color-success: #2abd36;
  --jrc-color-info: #20a9ff;
  --jrc-color-error: #dd2109;
  --jrc-color-warning: #ff8f20;
  --jrc-color-border: #d9d9d9;
}
```
